COMMENTS

I see Warning about LF and CRLF on windows. What do I do?

A: No need to worry too much about it. CRLF and LF are different ways Windows and Linux marks the end of a line (and the beginning of a new line). Git came from Linux, thus tries to convert the line-ending marks to the linux way (LF) on your windows computer (CRLF).

How do you make bash show you a “/” after directories when using “ls”?

Justin: Try typing “man ls” (if you’re on a Mac) to see some of the options you can give to the “ls” command - one of them is “-F”, so try “ls -F”. There are lots of other options too. (It looks like Daniel has his shell configured to do that automatically.)

Cheri: I can move my file “down” a directory, but how do I move it “up”? Like, now that it’s in the “workshop” directory, can I move it back to the Desktop?

Zhao: You can always use the absolute path of the file and directory. Also in Linux “..” is short for parent directory and “.” is short for current directory.

Justin: So as Zhao said, you’d do something like “mv myfile.txt ../myfile.txt”

Cheri: I have to do that from inside the workshop directory, yes?

Justin: Yes, so how about “mv workshop/myfile.txt myfile.txt”?
